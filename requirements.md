# Sistema de préstamo de bicicletas

## Stakeholders
Estudiantes 
Profesores
Vigilantes de seguridad
Administración Universitaria
Equipo de tecnología

## Requisitos funcionales

El sistema debe permitir a los usuarios consultar disponibilidad de bicicletas
El sistema debe permitir reservar una bicicleta y visualizar su información asociada, como el estado actual y los accesorios incluidos.
El sistema debe permitir al personal encargado  actualizar el estado de cada bicicleta despues del uso
El sistema debe permitir consultar las bicicletas en uso con el usuario 
El sistema debe permitir visualizar un mapa con las bicicletas cercanas disponibles 
El sistema debe permitir enviar notificaciones a los diferentes usuarios del sistema sobre cambios relevantes
El sistema debe permitir registrar reportes de bicicletas perdidas o robadas
## Requisitos no funcionales

El sistema debe mostrar la información solicitada por el usuario en menos de 1.5 segundos.
El sistema debe de visualizarse correctamente en los diferentes dispositivos
El sistema debe tener una interfaz fácil de usar para que los usuarios puedan realizar acciones de forma ágil
El sistema debe visualizar la información en tiempo real para garantizar la integridad de datos
